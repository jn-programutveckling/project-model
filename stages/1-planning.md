# Planning stage

This is the first stage of the project model. In this stage we set up the GitHub repository, write basic repository documents and plan out the development as much as possible in this early stage. As with all stages in the model, the planning stage isn't meant to be visited just once. Each stage should be seen as part of one iteration of development, not just part of the whole project life cycle.

1.   Create a new GitHub repository with issues enabled. Write a short description and optionally add tags.
2.   Add [priority labels](planning/labels.md). Remove GitHub's standard labels (or keep them, it doesn't really matter).
3.   Use the [templates](../templates/README.md) as a starting point for writing basic repository documentation. Don't worry if you don't yet know what to write in README. The sections *Installation* and *Usage* are much easier to write once you've made the first release. Most of the repo documents should be seen as living and be updated frequently throughout development.
4.   Create a [work breakdown structure](planning/wbs.md) and plan as much as possible. I use [Task Coach](https://www.taskcoach.org/), but other tools works too. The reasons I use Task Coach is that it supports sub-tasks, making it easy to group tasks in various levels of granularity; and it supports dependencies between tasks.
