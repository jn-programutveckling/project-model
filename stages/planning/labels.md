# Labels

When creating a GitHub repository and enables issues, a number of default labels are created. These work fine, but I think they are too many and are mostly geared toward projects with multiple collaborators and a lively community. Since I'm a lone developer, I don't need most of them. Instead I am more interested in what I call priority labels.

## Priority labels

| Name        | Description                                 | Colour  |
| ----------- | ------------------------------------------- | ------- |
| bug         | Bugs and problems with the code.            | #B60205 |
| docs        | Documentation, either internal or external. | #FBCA04 |
| improvement | Improvements on existing features.          | #0E8A16 |
| new feature | Work on new features.                       | #1D76DB |

These labels correspond to how issues are prioritised in the network model, as the following list (1 is highest priority):

1.   bug
2.   improvement
3.   docs
4.   new feature
