# Working stage

This is the second stage of the project model. In this stage we work on tasks in the work breakdown structure, perform testing and update documentation. As with all stages in the model, the working stage isn't meant to be visited just once. Each stage should be seen as part of one iteration of development, not just part of the whole project life cycle.

1.   For each task in the work breakdown structure…
     1.   Until the task is completed…
          1.   Write unit tests.
          2.   Write code so that the tests pass.
          3.   Refactor the code.
     2.   Update documentation of the code and the change-log, and if applicable, external documentation such as read-me, manual or wiki.

Remember to commit the code often.
